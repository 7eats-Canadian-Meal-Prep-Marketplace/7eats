# R2 Storage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a typed, per-bucket Cloudflare R2 storage layer in `lib/storage/` that the rest of the app can import to upload, retrieve, and delete files.

**Architecture:** A single `S3Client` singleton in `client.ts` is shared across six focused modules — one per bucket plus a shared constants file and a barrel export. Each bucket module exposes only the operations valid for that bucket (no public URL helpers on the private certs bucket, no signed URL helpers on the public CDN buckets), enforced by TypeScript's type system.

**Tech Stack:** `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner`, Vitest for unit tests, `vi.mock` for SDK mocking.

```typescript
import {
  GetObjectCommand,
  PutObjectCommand,
} from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";
import { r2Client } from "@/lib/storage/client";
import { BUCKETS } from "@/lib/storage/buckets";

export async function uploadCert(
  cookId: string,
  fileName: string,
  body: Buffer,
  contentType: string,
): Promise<string> {
  const key = `certs/${cookId}/${Date.now()}-${fileName}`;
  await r2Client.send(
    new PutObjectCommand({
      Bucket: BUCKETS.CERTS,
      Key: key,
      Body: body,
      ContentType: contentType,
    }),
  );
  return key;
}

export async function getSignedCertUrl(
  key: string,
  expiresIn = 900,
): Promise<string> {
  const clampedExpiry = Math.min(expiresIn, 3600);
  return getSignedUrl(
    r2Client,
    new GetObjectCommand({ Bucket: BUCKETS.CERTS, Key: key }),
    { expiresIn: clampedExpiry },
  );
}
```

```typescript
import {
  DeleteObjectCommand,
  PutObjectCommand,
} from "@aws-sdk/client-s3";
import { r2Client } from "@/lib/storage/client";
import { BUCKET_CONFIG, BUCKETS } from "@/lib/storage/buckets";

export async function uploadListingPhoto(
  listingId: string,
  fileName: string,
  body: Buffer,
  contentType: string,
): Promise<string> {
  const key = `listings/${listingId}/${Date.now()}-${fileName}`;
  await r2Client.send(
    new PutObjectCommand({

      Key: key,
      Body: body,
      ContentType: contentType,

  );

export function getListingPhotoUrl(key: string): string {

  return `${base}/${key}`;
}

  await r2Client.send(
    new DeleteObjectCommand({ Bucket: BUCKETS.LISTINGS, Key: key }),
  );
}
```

```typescript
import {
  CopyObjectCommand,
  DeleteObjectCommand,
  PutObjectCommand,
} from "@aws-sdk/client-s3";
import { r2Client } from "@/lib/storage/client";
import { BUCKETS, type TempDestBucket } from "@/lib/storage/buckets";

export async function uploadToTemp(
  key: string,
  body: Buffer,
  contentType: string,
): Promise<void> {
  await r2Client.send(
    new PutObjectCommand({
      Bucket: BUCKETS.TEMP,
      Key: key,
      Body: body,
      ContentType: contentType,
    }),
  );
}

export async function moveFromTemp(
  key: string,
  destBucket: TempDestBucket,
  destKey: string,
): Promise<void> {
  await r2Client.send(
    new CopyObjectCommand({
      Bucket: destBucket,
      CopySource: `${BUCKETS.TEMP}/${key}`,
      Key: destKey,
    }),
  );
  await r2Client.send(
    new DeleteObjectCommand({ Bucket: BUCKETS.TEMP, Key: key }),
  );
}

export async function deleteTemp(key: string): Promise<void> {
  await r2Client.send(
    new DeleteObjectCommand({ Bucket: BUCKETS.TEMP, Key: key }),
  );
}
```

**Bucket lifecycle TTL:** The 24h lifecycle rule on `homecook-uploads-temp` must be set manually in the Cloudflare R2 dashboard. There is no code step for this — it is a one-time infrastructure task.

**Bucket creation:** Run these commands once with Wrangler before testing against live R2:
```bash

wrangler r2 bucket create homecook-listings-public
wrangler r2 bucket create homecook-avatars-public
wrangler r2 bucket create homecook-uploads-temp
```

**Key format convention:**
- Certs: `certs/{cookId}/{timestamp}-{fileName}`
- Listing photos: `listings/{listingId}/{timestamp}-{fileName}`
- Avatars: `avatars/{userId}/{timestamp}-{fileName}`
- Temp: caller-supplied key (the API route that calls `uploadToTemp` owns the key format)
